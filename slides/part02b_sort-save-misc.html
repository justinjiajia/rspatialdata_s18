Sorting Vectors

There are two approaches to sorting:

sort(x): Change the order of the elements in vector

order(x): Leave x alone, but find the order the elements are in and get their indices in that order.

To illustrate, let's make a vector of random values.

x <- round(1000 * runif(10))
x
##  [1] 831 186 803 986 687 460 802 932 965 635
sort(x)
##  [1] 186 460 635 687 802 803 831 932 965 986
order(x)
##  [1]  2  6 10  5  7  3  1  8  9  4

If you compute the order of the elements, you can return a sorted copy of the list by putting the indices inside the square brackets.

idx <- order(x)
x[idx]
##  [1] 186 460 635 687 802 803 831 932 965 986

Sorting in descending order can be done with the decreasing argument.

idx <- order(x, decreasing=TRUE)
x[idx]
##  [1] 986 965 932 831 803 802 687 635 460 186

Picking a Random Sample

We've already seen examples of generating random numbers with functions like runif() and rnorm(). Another common task is selecting a random sample. Suppose you want to pick three letters from the alphabet at random (for a password for example).

sample(LETTERS, size=3)
## [1] "A" "M" "W"

Another way to pick a sample with the sample() function is to generate a vector of logical values, and then use that as the expression for the rows to return.

control_grp_yn <- sample(c(T,F), size=26, replace=TRUE)
control_grp_yn
##  [1] FALSE FALSE FALSE  TRUE FALSE  TRUE FALSE  TRUE FALSE FALSE  TRUE
## [12]  TRUE  TRUE  TRUE  TRUE  TRUE  TRUE FALSE FALSE FALSE FALSE FALSE
## [23]  TRUE  TRUE  TRUE  TRUE
LETTERS[control_grp_yn]
##  [1] "D" "F" "H" "K" "L" "M" "N" "O" "P" "Q" "W" "X" "Y" "Z"

Dealing with Missing, Undefined, and Invalid Values

An object may have a missing or undefined value if it hasn't been initialized yet, a value wasn't passed, an operation couldn't work.

R has two resevered words it uses to note missing values.

NULL - undefined
NA - missing

These are not the same as errors!

NULL can not exist in a vector, but NA can.

x <- c(rep(NA, 10))
x
##  [1] NA NA NA NA NA NA NA NA NA NA
y <- c(rep(NULL, 10))
y
## NULL

Testing for NA and NULL:

is.na(x)
is.null(x)

Other special values

Empty Strings

x == ""

Inf and -Inf

5/0
## [1] Inf

Lists

A List object is an ordered group of data that are not of the same type. Lists are similiar to vectors except that,

  • the elements can be referenced by name as well as index
  • elements can be different types
  • elements can anything (including other lists!)
  • instead of using c() like in vector creation, use list() to create a list
move_data <- list(id="toni", species="buffalo", age=12, sex="female", selected=TRUE, wt=182, herd=c("west", "hilltop"))
move_data
## $id
## [1] "toni"
## 
## $species
## [1] "buffalo"
## 
## $age
## [1] 12
## 
## $sex
## [1] "female"
## 
## $selected
## [1] TRUE
## 
## $wt
## [1] 182
## 
## $herd
## [1] "west"    "hilltop"

You grab individual elements with the $ character or double square brackets. If you're not sure what the element names are, you can check with names() (and change them also).

names(move_data)
## [1] "id"       "species"  "age"      "sex"      "selected" "wt"      
## [7] "herd"
move_data$id
## [1] "toni"
move_data[[3]]
## [1] 12
move_data[["age"]]
## [1] 12

Note $ and [[ ]]] grab an individual element. Use single square brackets to return a sub-list.

move_data[ c(3,5)]
## $age
## [1] 12
## 
## $selected
## [1] TRUE

Reassign a list element

move_data$selected <- FALSE
move_data$selected
## [1] FALSE

To add a list element, just assign it a value and R will add it 'on the fly'.

move_data$mother <- "luci"

To delete a list element, set it to NULL.

move_data$herd <- NULL
names(move_data)
## [1] "id"       "species"  "age"      "sex"      "selected" "wt"      
## [7] "mother"

S4 Objects

S4 objects are like 'super lists'.

In addition to list elements, they can have 'slots' that you grab with the @ character.

Spatial data classes from the sp package are S4 objects.

Saving Your Work

A big reason for using R is to be able to reproduce your work.

You can save both scripts and variables to files on disk.

Save scripts in RStudio as *.R files.

Directories

When you import or save objects to disk using a command, you don't have the luxury of an interactive file dialog. You have to know where it's going.

All file paths are relative to the working directory. You can view or set the working directory from RStudio or with commands at the console:

setwd("/projects/soc101/sf_census/")
getwd()
## [1] "C:/projects/soc101/sf_census"

R uses standard Linux notion when interpreting directory names. The default 'Home' directory is usually your Documents folder, and can be specified with the ~ character.

path.expand("~")
## [1] "C:/Users/Andy/Documents"

The 'root' directory on the current volume (drive) can be specified with the '/' character. You can specify the drive letter if you'd like also (e.g., c:/).

path.expand("/")
## [1] "/"

Note for Windows users: R requires forward slashes (/), not back slashes (\). Sorry about that. If you really want to keep using back slashes, you can use a double back slash (\\)

You can specify the current directory with a period (.). If you want to save something to a sub-directory of the active directory, start with the period and then keep going.

setwd("/projects/soc101/sf_census")
getwd()
## [1] "C:/projects/soc101/sf_census"
setwd("./data")
getwd()
## [1] "C:/projects/soc101/sf_census/data"

Saving Objects to Disk

You can save individual objects (variables) to disk with the save() function. If you don't specify a directory as part of the file name, it will go in the working directory.

x_nums <- rnorm(1000) * 20
## Save to home directory
save(x_nums, file="~/first_try.RData")

.RData and .Rda are common extensions for R data files, but you can name your file anyting you want.

The save() function saves objects in a binary, compressed file format that R understands. This makes it very flexible and easy to bring back into R using the sister function load().

To save your entire workspace (all variables in memory), you can use the save.image() function. RStudio also have a 'Save Workspace' button on the 'Environment' window, and will probably ask if you want to save your workspace when you quit the program.

Other ways to save data

If you want to export data to use with another program, you probably want to use a different function that saves in a more standard file format, like write.csv() or write.table().

Use the sink() function to redirect output from the console to a file. Run sink() again without an argument to resume sending output to the console.

Spatial objects can be written to GIS formats with functions from the rgdal package, including writeGDAL (vector) and writeGDAL (raster).

Plots can be saved as PNG and PDF files.

Loading Objects from Disk

To load a file back into memory, use the load() function.

rm(x_nums)   ## remove x_nums from memory
load("~/first_try.RData")
summary(x_nums)
##     Min.  1st Qu.   Median     Mean  3rd Qu.     Max. 
## -60.9022 -13.5849   0.6754   0.3599  13.2205  66.2261

Short Cuts

If you're working at the console and want to quickly get the name to a file, you can use the file.choose() function.

In RStudio, you can use the Files tab to browse your files and set the working directory.

Saving Scripts

You can save and open scripts, R Markdown files, etc. from RStudio.

If you have a set of commands that you want to rerun repeatedly, you can save them as a *.R file and then run them all at once with source() function.

source("import_clean_data.R")
